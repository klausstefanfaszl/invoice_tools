1. Überblick

invoice_tools.exe ist ein kombiniertes Kommandozeilenprogramm mit drei integrierten Tools:

ToolFunktion
extractorFelder aus PDFs extrahieren (Lieferant, Beträge, Datum, IBAN …)
inboxRechnungsanhänge automatisch aus E-Mail-Postfach laden und ablegen
mailtoPDF-Ausgangsrechnungen automatisch per E-Mail versenden

Grundsyntax:

invoice_tools.exe extractor [Optionen] PDF [PDF ...]
invoice_tools.exe inbox     -m MODUS [Optionen]
invoice_tools.exe mailto    --rg-nr N | --min-rg-nr N [Optionen]

Alle drei Tools teilen das _internal-Verzeichnis. Für den Betrieb sind keine weiteren Installationen erforderlich.

3. Tool: extractor

Extrahiert strukturierte Daten (Lieferant, Rechnungsnummer, Beträge, Datum, IBAN, Zahlungsart) aus PDF-Rechnungen. Unterstützt ZUGFeRD/Factur-X, KI-Analyse und Regex-Fallback.

3.1 Extraktionsprioritäten

PrioritätMethodeVoraussetzung
1 (höchste)ZUGFeRD/Factur-X XMLEingebettetes XML im PDF vorhanden
2KI-AnalyseAPI-Key in der Config konfiguriert
3 (Fallback)Regex-ExtraktionDurchsuchbarer Text im PDF
–OCR (Tesseract)Tesseract installiert + gescanntes PDF

3.2 Parameter

ParameterKurzStandardBeschreibung
--config DATEI-cinvoice_extractor_config.xmlXML-Konfigurationsdatei
--format FORMAT-fstdoutstdout · txt · pdf · csv · json · xml
--output DATEI-o(automatisch)Ausgabedatei. -o STDOUT leitet auf die Konsole um.
--logfile DATEI-l(kein Log)Verarbeitungsprotokoll in Datei schreiben
--debug LEVEL-d00=aus · 1=Pfad · 2=Details · 3=Vollausgabe
--api DATEI-a(automatisch)Zentrale KI-API-Konfigurationsdatei

3.3 Ausgabeformate

FormatBeschreibung
stdoutTabellarische Ausgabe direkt auf die Konsole (Standard)
txtIdentisch mit stdout, in Datei gespeichert
pdfFormatierte PDF-Tabelle mit Spaltenbezeichnungen und optionaler Summenzeile

3.4 Beispielaufrufe

Einzelne Rechnung auf Konsole:

invoice_tools.exe extractor rechnung.pdf

Alle PDFs eines Monats als PDF-Bericht mit Log:

invoice_tools.exe extractor -c invoice_extractor_config_RA.xml ^
    -f pdf -o Rechnungsausgang_01.pdf -l log.log ^
    "C:\Buchhaltung\2026\01\*.pdf"

CSV-Ausgabe für Excel:

invoice_tools.exe extractor -f csv -o ausgabe.csv "*.pdf"

4. Tool: inbox

Verbindet sich mit einem Exchange- oder IMAP-Postfach, lädt ungelesene E-Mails mit PDF-Anhängen, extrahiert Rechnungsfelder und speichert die PDFs automatisch in der konfigurierten Verzeichnisstruktur. Nicht-Rechnungen werden übersprungen.

4.1 Parameter

ParameterKurzStandardBeschreibung
--modus MODUS-m(Pflicht)dry · unread · all · archiv (siehe 4.2)
--config DATEI-cinvoice_inbox_config.xmlXML-Konfigurationsdatei
--debug LEVEL-d00=aus · 1=Pfad · 2=Details · 3=Vollausgabe
--log DATEI-l(kein Log)Protokolldatei; ohne -d wird stdout vollständig dorthin umgeleitet
--dry-run––Simulation, kombinierbar mit -m all oder -m unread
--bzv MODUS-b–BankingZV-Export: dry=Anzeige · json=+JSON-Datei · export=+BankingZV-Aufruf
--bdir VERZ.-B–Basisverzeichnis für PDF-Ablage; überschreibt <BaseDir> aus der Config
--export-excel-e–Rechnungsdaten in Excel-Eingangstabelle schreiben (Duplikat-Prüfung aktiv)
--api DATEI-a(automatisch)Zentrale KI-API-Konfigurationsdatei

4.2 Modi

ModusSpeichernAls gelesenArchivierenWelche Mails
dryNeinNeinNeinNur ungelesene — zeigt was gespeichert würde
unreadJaJaNeinNur ungelesene Mails
allJaJaNeinAlle Mails (gelesen + ungelesen)
archivJaJaJaNur ungelesene — verschiebt verarbeitete Mails in den Archiv-Ordner

Archiv-Ordner konfigurieren: <ArchiveFolder>Archiv</ArchiveFolder> im <Mailbox>-Block der invoice_inbox_config.xml.

4.3 Unterstützte Postfach-Typen

TypProtokollTypische AnbieterPflichtfelder

Hinweis: Gmail: Bei aktivierter 2-Faktor-Authentifizierung ist ein App-Passwort erforderlich.

4.4 Dateiablage

PDFs werden automatisch in der folgenden Struktur abgelegt:

<BaseDir>/<YYYY>/<MM>/ER_MMTT_<Lieferant>_<RechnungsNr>.pdf

Rechnungen ohne erkennbares Datum landen in <BaseDir>/_unbekannt/. Vorhandene Dateien werden nicht überschrieben — bei Namenskollision wird ein Suffix angehängt (_2, _3, …).

4.5 Filter

FilterConfig-ElementFunktion
Dateinamen-Filter<AttachmentFilter><SkipPattern>Anhänge überspringen, deren Dateiname das Muster enthält (z.B. "summary", "lastschrift")
Pflichtfeld-Filter<InvoiceFilter><RequiredField>PDF wird nur gespeichert, wenn alle Pflichtfelder extrahiert wurden (z.B. InvoiceNumber, GrossAmount)

4.6 Excel-Export (--export-excel / -e)

Mit dem Flag -e wird jede erfolgreich verarbeitete Rechnung als neue Zeile in eine Excel-Datei geschrieben. Die Zieltabelle wird über den Tabellennamen (Standard: tb_rechnungen) referenziert — Spaltenreihenfolge und -namen werden dynamisch aus der Tabellendefinition ermittelt.

Befüllte Spalten (konfigurierbar per <Spaltenmapping>):

Excel-SpalteQuelleTyp
Re-DatumInvoiceDatedatum
Name/LieferantSupplierNameText
RE-NrInvoiceNumberText
TYP{StandardTyp}Text (aus Config)
NettoNetAmountbetrag
BruttoGrossAmountbetrag
Fällig_amDueDatedatum
IBANIBANiban (erstes Element)
Konto-ID{KontoId}Text (aus BankingZV-Routing)

Spalten mit calculatedColumnFormula (z.B. Monat, Jahr, Steuer) werden automatisch erkannt und aus der Vorgängerzeile kopiert.

Duplikat-Prüfung: Vor dem Schreiben wird die konfigurierte Duplikat-Spalte (Standard: RE-Nr) geprüft. Ein bereits vorhandener Eintrag erzeugt einen Info-Hinweis — kein Fehler, kein Abbruch.

Dry-Run-Vorschau: Mit -m dry -e -d 1 werden die Zellwerte aller zu schreibenden Spalten angezeigt, ohne die Datei zu verändern.

Config-Block <ExcelExport> in invoice_inbox_config.xml:

<ExcelExport>
  <DateiPfad>C:\Pfad\zur\Rechnungseingang.xlsx</DateiPfad>
  <Tabellenblatt>ER</Tabellenblatt>
  <Tabellenname>tb_rechnungen</Tabellenname>
  <StandardTyp>E</StandardTyp>
  <DuplikatSpalte>RE-Nr</DuplikatSpalte>
  <Spaltenmapping>
    <!-- name=Excel-Spalte  typ=datum|betrag|iban|(leer=Text) -->
    <Spalte name="Re-Datum"       typ="datum" >InvoiceDate</Spalte>
    <Spalte name="Name/Lieferant"             >SupplierName</Spalte>
    <Spalte name="RE-Nr"                      >InvoiceNumber</Spalte>
    <Spalte name="TYP"                        >{StandardTyp}</Spalte>
    <Spalte name="Netto"          typ="betrag">NetAmount</Spalte>
    <Spalte name="Brutto"         typ="betrag">GrossAmount</Spalte>
    <Spalte name="Faellig_am"     typ="datum" >DueDate</Spalte>
    <Spalte name="IBAN"           typ="iban"  >IBAN</Spalte>
    <Spalte name="Konto-ID"                   >{KontoId}</Spalte>
  </Spaltenmapping>
</ExcelExport>

Hinweis: Fehlt <Spaltenmapping>, wird das eingebaute Standard-Mapping verwendet. Die Tabelle wird nach jedem Einfügen automatisch auf die neue Zeile ausgedehnt.

4.7 Beispielaufrufe

invoice_tools.exe inbox -m dry
invoice_tools.exe inbox -m unread -c invoice_inbox_config.xml
invoice_tools.exe inbox -m archiv -b export
invoice_tools.exe inbox -m unread -b export -l inbox.log
invoice_tools.exe inbox -m dry -e -d 1              (Vorschau Excel-Zellwerte)
invoice_tools.exe inbox -m unread -e                (PDF ablegen + Excel schreiben)
invoice_tools.exe inbox -m unread -e -b export      (PDF + Excel + BankingZV)
invoice_tools.exe inbox -m dry -B "F:\Buchhaltung"  (BaseDir per Parameter)
invoice_tools.exe inbox -m unread -B "F:\Buchhaltung" -e

Mit -l inbox.log und ohne -d wird kein Text auf stdout ausgegeben. Fehlermeldungen (stderr) bleiben immer sichtbar. Der Exit-Code steht dem Batch-Skript über %errorlevel% zur Verfügung.

5. Tool: mailto

Versendet PDF-Ausgangsrechnungen automatisch per E-Mail. Das Tool durchsucht ein konfiguriertes Verzeichnis nach PDF-Dateien, extrahiert aus dem Dateinamen die Rechnungsnummer, ermittelt die Empfängeradresse auf einem der vier Wege und sendet die PDF als Anhang mit einem standardisierten HTML-Mailtext.

Vor dem Versand wird standardmäßig eine Bestätigung eingeholt. Mit --noconfirm entfällt diese. Im --dry-run-Modus wird nichts gesendet.

5.1 Parameter

ParameterKurzStandardBeschreibung
--rg-nr N-r–Einzelne Rechnungsnummer (Pflicht, alternativ zu --min-rg-nr)
--min-rg-nr N––Kleinste zu versendende Rechnungsnummer (Pflicht, alternativ zu --rg-nr)
--max-rg-nr N––Größte zu versendende Rechnungsnummer (nur zusammen mit --min-rg-nr)
--adress-modus M-a0Adressermittlung: 0=Test-Adresse, 1=PDF-Text, 2=DB, 3=PDF+DB (s. 5.2)
--rg-datum DATUM––Nur PDFs mit diesem Dateidatum versenden. Formate: TT.MM · TT.MM.JJ · TT.MM.JJJJ. Bei TT.MM wird das aktuelle Jahr verwendet.
--dry-run––Simulation: zeigt was gesendet würde, ohne tatsächlich zu senden
--noconfirm––Mails sofort versenden ohne Bestätigungsdialog
--debug LEVEL-d00=aus · 1=Überblick · 2=Details
--config DATEI-cinvoice_tools_mailto_config.xmlXML-Konfigurationsdatei

Hinweis: Entweder --rg-nr oder --min-rg-nr muss angegeben werden. Die beiden Parameter schließen sich gegenseitig aus.

5.2 Adressmodi

ModusQuelleBeschreibung
0 (Standard)KonfigurationsdateiFeste Test-Adresse aus <TestAdresse> — alle Mails gehen an dieselbe Adresse. Ideal für Tests.
1PDF-TextSucht nach dem konfigurierten Marker (Standard: email:) im selektierbaren Text der PDF. Beispiel im PDF: E-Mail: kunde@example.com
2MySQL-DatenbankFührt bis zu zwei SQL-Abfragen aus. SQL2 wird nur verwendet, wenn SQL1 kein Ergebnis liefert. Die erste Spalte des Ergebnisses ist immer die E-Mail-Adresse; eine optionale zweite Spalte kann die Auftragsnummer enthalten (siehe 5.4).
3PDF-Text + DBVersucht zuerst Modus 1 (PDF-Text), bei Misserfolg Modus 2 (Datenbank).

5.3 SMTP-Konfiguration

ElementBeschreibung
<Host>SMTP-Servername oder IP-Adresse
<Port>Port: 25 (plain), 587 (STARTTLS), 465 (SMTPS)
<SSL>true = direktes SMTPS (Port 465), false = STARTTLS oder plain
<TLS>true = STARTTLS verwenden; false = plain SMTP (z.B. internes Relay Port 25). Wird automatisch auf false gesetzt wenn Port=25 oder SSL=true.
<User>Benutzername für SMTP-Login; leer lassen bei anonymem Relay (kein Login)
<Password>Passwort für SMTP-Login
<Absender>From-Adresse, z.B. Firma GmbH <rechnung@firma.de>
<AntwortAn>Optionale Reply-To-Adresse; leer lassen wenn nicht benötigt

Minimal-Konfiguration für internes Relay (Port 25, kein Login):

<SMTP>
  <Host>mailserver.intern</Host>
  <Port>25</Port>
  <SSL>false</SSL>
  <TLS>false</TLS>
  <User></User>
  <Password></Password>
  <Absender>UHDE Datentechnik GmbH <rechnung@uhde.de></Absender>
</SMTP>

5.4 Datenbankabfrage (MySQL)

Für Adressmodus 2 und 3 kann eine MySQL-Datenbank konfiguriert werden. Der Platzhalter {rgnr} in den SQL-Abfragen wird durch die Rechnungsnummer (Integer) ersetzt. Die erste Spalte des Abfrageergebnisses wird immer als E-Mail-Adresse verwendet.

Gibt die SQL-Abfrage eine zweite Spalte zurück, wird deren Wert als Auftragsnummer interpretiert. Das Tool sucht dann im PDF-Verzeichnis nach weiteren PDF-Dateien, deren Dateiname die letzten X Ziffern dieser Auftragsnummer enthält (X konfigurierbar über <AuftragNrStellen>). Gefundene Auftrags-PDFs werden der E-Mail als zusätzliche Anhänge beigefügt.

<Datenbank>
  <Host>localhost</Host>
  <Port>3306</Port>
  <DbName>meine_datenbank</DbName>
  <User>dbbenutzer</User>
  <Password>passwort</Password>
  <!-- Primärabfrage; Spalte 2 = Auftragsnummer (optional) -->
  <SQL1>SELECT email, auftragsnr FROM auftraege WHERE Nummer*1 = {rgnr}</SQL1>
  <!-- Fallback-Abfrage (optional) -->
  <SQL2>SELECT email FROM kunden WHERE kunden_nr =
        (SELECT kunden_nr FROM auftraege WHERE Nummer*1 = {rgnr})</SQL2>
  <!-- Letzte X Stellen der Auftragsnummer für PDF-Suche; 0 = deaktiviert -->
  <AuftragNrStellen>5</AuftragNrStellen>
</Datenbank>

Beispiel: Auftragsnummer AUF-2026-01234, <AuftragNrStellen>5</AuftragNrStellen> → Das Tool sucht nach Dateien, die 01234 im Dateinamen enthalten (z.B. Auftragsbestätigung_01234.pdf). Die Rechnungs-PDF selbst wird dabei übersprungen.

Standardmäßig wird im <PdfVerzeichnis> gesucht. Mit dem optionalen Element <PdfVerzeichnis_Auftraege> kann ein separates Verzeichnis für Auftrags-PDFs angegeben werden:

<PdfVerzeichnis>C:\Rechnungsausgang</PdfVerzeichnis>
<PdfVerzeichnis_Auftraege>C:\Auftraege</PdfVerzeichnis_Auftraege>

Installation mysql-connector-python: py -m pip install mysql-connector-python. Ist das Paket nicht installiert, gibt das Tool eine Warnung aus und überspringt die Datenbankabfrage.

6.3 Mailto-Konfiguration (invoice_tools_mailto_config.xml)

Vollständiger Aufbau der Konfigurationsdatei:

<MailtoSenderConfig>

  <SMTP>
    <Host>smtp.firma.de</Host>
    <Port>587</Port>
    <SSL>false</SSL>
    <TLS>true</TLS>          <!-- false für internes Relay Port 25 -->
    <User>rechnung@firma.de</User>
    <Password>...</Password>
    <Absender>Firma GmbH &lt;rechnung@firma.de&gt;</Absender>
  </SMTP>

  <!-- Verzeichnis mit den zu versendenden Ausgangsrechnungs-PDFs -->
  <PdfVerzeichnis>C:\Rechnungsausgang</PdfVerzeichnis>

  <!-- Regex für Rechnungsnummer im Dateinamen; named group "rgnr" bevorzugt -->
  <RgNrRegex>(?P&lt;rgnr&gt;\d+)</RgNrRegex>

  <MailVorlage>
    <Betreff>Rechnung Nr. {rgnr}</Betreff>
    <HtmlBody><![CDATA[ ... HTML-Mailinhalt ... ]]></HtmlBody>
  </MailVorlage>

  <!-- Adressmodus 0: feste Test-Adresse -->
  <TestAdresse>test@firma.de</TestAdresse>

  <!-- Suchbegriff für Adressmodus 1 -->
  <EmailTextMarker>email:</EmailTextMarker>

  <!-- Adressmodus 2 / 3: MySQL-Datenbank (optional) -->
  <Datenbank>
    <Host>localhost</Host>
    <Port>3306</Port>
    <DbName>meine_db</DbName>
    <User>root</User>
    <Password>...</Password>
    <SQL1>SELECT email, auftragsnr FROM auftraege WHERE Nummer*1 = {rgnr}</SQL1>
    <!-- 0 = Auftrags-PDF-Suche deaktiviert -->
    <AuftragNrStellen>0</AuftragNrStellen>
  </Datenbank>

</MailtoSenderConfig>

8. Exit-Codes

Alle Tools melden ihren Verarbeitungsstatus als Exit-Code an das aufrufende Skript.

8.1 inbox

CodeKonstanteBedeutung
0OKErfolgreich — alle Rechnungen gespeichert (inkl. keine Mails gefunden)
1CONFIGKonfigurationsfehler — Config-Datei fehlt, ungültig oder Pflichtfeld leer
2VERBINDUNGVerbindungsfehler — Postfach nicht erreichbar oder Netzwerkfehler
3KI-ABBRUCHKI-Abbruch — alle konfigurierten KI-Provider nicht verfügbar
4TEILERFOLGTeilerfolg — mindestens 1 Rechnung gespeichert, aber mindestens 1 Fehler
5ALLE FEHLAlle Anhänge fehlgeschlagen — Mails vorhanden, aber 0 Rechnungen gespeichert

8.2 extractor

CodeBedeutung
0Erfolgreich — mind. 1 PDF verarbeitet
1Fehler — Konfiguration ungültig, keine PDFs gefunden oder Ausgabefehler

8.3 mailto

CodeKonstanteBedeutung
0OKErfolgreich — alle Mails gesendet (inkl. keine PDFs gefunden / Dry-Run)
1CONFIGKonfigurationsfehler
2VERBINDUNGSMTP-Verbindungsfehler
4TEILERFOLGMindestens 1 Mail gesendet, mindestens 1 Fehler
5ALLE FEHLKeine einzige Mail erfolgreich gesendet

Tipp für Batch-Skripte: if %errorlevel% NEQ 0 ... prüft ob ein Fehler vorliegt. Für differenzierte Auswertung: if %errorlevel% EQU 4 ... usw.

9. IBAN-Validierung

Alle extrahierten IBANs werden automatisch nach ISO 13616 (Modulo-97-Verfahren) validiert. Ungültige oder maskierte IBANs (mit * oder XXX) werden stillschweigend verworfen.
